홈택스 관련 시트 제어 및 계산서 발행을 처리하는 스킬이야.

> **🚀 처음 사용하는 환경이라면**: 같은 폴더의 `heyclaude.md`를 따라 환경 세팅을 먼저 진행. 사전 요구사항(module_auth, module_api_key/.env, Google API 패키지) 외에도 본인 회사 상호와 시트 정보를 스크립트에 입력해야 동작해.

## 기능 구성

| 기능 | 스크립트 |
|------|---------|
| 세금계산서(발행) 시트 CRUD | `tax_invoice.py` |
| 거래처 시트 CRUD | `vendor.py` |
| 실제 계산서 발행 | `issue_invoice` (본인 환경에서 별도 구성) |

**스크립트 디렉토리**: `~/.claude/skills/hometax_manager/scripts/`

> ⚠️ 처음 사용 시 두 스크립트 상단의 `SUPPLIER`(본인 회사 상호), `SPREADSHEET_ID`(본인 시트 ID), `SHEET_INVOICE`/`SHEET_VENDOR`/`SHEET_NAME`(탭 이름) 값을 채워야 동작함.

## 1. 세금계산서(발행) 시트

### 열 구조 (예시 — 본인 시트 구조에 맞게 조정)

| 열 | 항목 | 입력 여부 | 비고 |
|----|------|----------|------|
| A | 공급자 상호 | ✅ 입력 | `SUPPLIER` 변수값 (본인 회사 상호) |
| B | 별칭(공급받는자/거래처) | ✅ 입력 | 거래처 A열 드롭다운 |
| C | 상호명 | 수식 자동 | B 기준 VLOOKUP |
| D | 사업자등록번호 | 수식 자동 | B 기준 VLOOKUP |
| E | 이메일 | 수식 자동 | B 기준 VLOOKUP |
| F | 작성일자(공급 연월일) | ✅ 입력 | YY-MM-DD |
| G | 월 | ✅ 입력 | 작성일의 월 |
| H | 일 | ✅ 입력 | **실제 공급일** (작성일과 다를 수 있음) |
| I | 합계금액(VAT포함 금액) | ✅ 입력 | = N + O 계산해서 입력 |
| J | 품목명 | ✅ 입력 | |
| K | 규격 | 미입력 | |
| L | 수량 | ✅ 입력 | 기본 1 |
| M | 단가 | ✅ 입력 | 공급가액 ÷ 수량 |
| N | 공급가액 | ✅ 입력 | 부가세 제외 금액 |
| O | 세액 | ✅ 입력 | 공급가액 × 10% 또는 직접 지정 |
| P | 비고 | 미입력 | |
| Q | 청구/영수 | ✅ 입력 | 기본 영수 |
| R | 발급하기(y) | 미입력 | 발급 실행파일이 처리 |
| S | 비고2 | 미입력 | |

### 세금계산서 입력 처리 절차

사용자가 "세금계산서 발행 준비해줘" 또는 계산서 입력을 요청하면:

**Step 1. 거래처 별칭 목록 먼저 조회**
```bash
cd ~/.claude/skills/hometax_manager/scripts && \
python3 tax_invoice.py --list-aliases
```
→ 거래처 시트 A열의 전체 별칭 목록 확인

**Step 2. 사용자 말과 별칭 매칭**
- 출력된 목록에서 사용자가 언급한 거래처와 가장 부합하는 별칭 선택
- 정확히 판단 안 될 경우 사용자에게 확인
- 찾은 별칭을 B열 값으로 사용

**Step 3. 나머지 정보 파악**

| 항목 | 열 | 필수 | 기본값 |
|------|----|----|--------|
| 거래처 별칭 | B | ✅ | Step 2에서 매칭 |
| 작성일자 | F | ✅ | **반드시 사용자에게 확인** |
| 공급월 | G | ✅ | 작성일자의 월 |
| 공급일(일) | H | ✅ | 작성일자의 일 (기간 있으면 종료일) |
| 합계금액(=N+O) | I | ✅ | 자동계산 |
| 품목명 | J | ✅ | 사용자 입력 |
| 수량 | L | ✅ | 1 |
| 단가 | M | ✅ | 공급가액 ÷ 수량 |
| 공급가액 | N | ✅ | 사용자 입력 (부가세 제외) |
| 세액 | O | ✅ | 공급가액 × 10% (또는 직접 지정) |
| 청구/영수 | Q | ✅ | 영수 (청구라고 하면 청구) |

### 금액 계산 규칙

**부가세 제외 (= VAT 미포함)** → 전달 금액이 단가(=공급가액)
| 열 | 값 |
|----|----|
| M 단가 | 전달 금액 |
| N 공급가액 | 단가와 동일 |
| O 세액 | 단가 × 10% |
| I 합계금액 | 단가 + 세액 |

**부가세 포함 (= VAT 포함)** → 전달 금액이 합계금액(I열)
| 열 | 값 |
|----|----|
| I 합계금액 | 전달 금액 |
| M 단가 | 합계금액 ÷ 1.1 (소수점 버림) |
| N 공급가액 | 단가와 동일 |
| O 세액 | 합계금액 - 단가 |

**검수 규칙 (입력 전 반드시 확인)**:
- 단가 = 공급가액 ✅
- 단가 + 세액 = 합계금액(I열) ✅

### 날짜 규칙
- F열: YY-MM-DD 형식 (예: 26-03-03) → **항상 사용자에게 공급연월일 확인**
- G열: F열의 월
- H열: F열의 일 (서비스 기간 있으면 종료일, 예: "3/1~3/18" → H=18)

### 입력 전 확인 절차

**1. 공급연월일(작성일자) 반드시 사용자에게 확인**
> "공급연월일(작성일자)이 어떻게 되나요?"

**2. Q열(청구/영수) 반드시 사용자에게 확인**
> "'영수'로 발행할까요, '청구'로 발행할까요?"

**3. 최종 내용 정리 후 확인받기**
```
아래 내용으로 입력할게요. 확인해주세요.

  거래처   : [별칭]
  작성일자 : [날짜]  (공급월: X월 X일)
  품목명   : [품목]
  수량     : [수량]
  단가     : [금액]원
  공급가액 : [금액]원  ← 단가와 동일
  세액     : [금액]원
  합계금액 : [금액]원  ← 단가 + 세액
  청구/영수: [구분]
```
→ 사용자가 "등록해줘" / "맞아" 등 확인하면 그때 스크립트 실행

### 명령어

**입력 (새 계산서)**
```bash
cd ~/.claude/skills/hometax_manager/scripts && \
python3 tax_invoice.py \
  --alias "[별칭]" \
  --item "[품목명]" \
  --amount [공급가액] \
  --vat [세액] \
  --billing [영수|청구] \
  [--supply-day 공급일] \
  [--date YYYY-MM-DD] \
  [--qty 수량]
```
※ --amount는 항상 공급가액(단가), --vat는 항상 세액을 전달

**수정**
```bash
python3 tax_invoice.py --edit --row [행번호] \
  [--item "수정품목"] [--amount 수정금액] [--vat 수정세액] [--billing 청구|영수]
```

**삭제**
```bash
python3 tax_invoice.py --delete --row [행번호]
```

**조회**
```bash
python3 tax_invoice.py --list           # 발행 목록
python3 tax_invoice.py --list-aliases   # 거래처 별칭 목록
```

## 2. 거래처 시트

### 열 구조

| 열 | 항목 |
|----|------|
| A | 별칭(구분) |
| B | 상호 |
| C | 사업자등록번호 |
| D | 공급자_성명(대표자) |
| E | 담당자명 |
| F | 담당자메일 |
| G | 사업장소재지 |
| H | 업태 |
| I | 종목 |

### 거래처 추가 절차

**⚠️ 별칭(A열) 반드시 사용자에게 확인받을 것**

사용자가 거래처 추가를 요청하면:
1. 전달받은 정보 정리 (상호, 사업자번호, 담당자 등)
2. **별칭을 어떻게 할지 사용자에게 확인**: "별칭(구분)을 어떻게 설정할까요?"
3. 확인 후 스크립트 실행

```bash
cd ~/.claude/skills/hometax_manager/scripts && \
python3 vendor.py --add \
  --alias "[확인받은 별칭]" \
  --company "[상호]" \
  --biz-num "[사업자번호]" \
  [--rep "[대표자]"] \
  [--contact "[담당자명]"] \
  [--email "[담당자메일]"] \
  [--address "[사업장소재지]"] \
  [--biz-type "[업태]"] \
  [--biz-item "[종목]"]
```

### 기타 명령어

**조회**
```bash
cd ~/.claude/skills/hometax_manager/scripts && python3 vendor.py --list
```

**수정**
```bash
python3 vendor.py --edit --alias "[별칭]" [수정할 옵션들...]
```

**삭제**
```bash
python3 vendor.py --delete --alias "[별칭]"
```

## 사용자 요청 파싱 가이드

| 사용자 말 | 처리 |
|-----------|------|
| "X 거래처 계산서 발행해줘" | tax_invoice.py 입력 |
| "거래처 목록 보여줘" / "어떤 거래처 있어" | vendor.py --list |
| "계산서 목록 보여줘" | tax_invoice.py --list |
| "7행 품목 수정해줘" | tax_invoice.py --edit --row 7 |
| "새 거래처 추가해줘" | vendor.py --add |

**공급일 주의**: 사용자가 "서비스 기간이 12/1~12/18이야"라고 하면 --supply-day 18. 별도 언급 없으면 작성일과 동일.

**별칭 매칭**: 부분 입력 허용. 복수 매칭 시 사용자에게 선택 요청.
