## AI스마트광고 vs 매출최적화광고 비교

| | AI스마트광고 | 매출최적화광고 |
|--|------------|--------------|
| 광고 대상 | 판매 중인 전체 상품 중 AI 자동 선택 | 내가 직접 선택한 상품만 |
| 탐색 능력 | 지금 고객이 찾는 상품을 AI가 자동 발견 | 등록 안 한 상품은 기회 놓침 |
| 품절 상품 | 자동 제외 | 광고 자동 중단 |
| 신규 상품 | 자동 포함 (초반 낮은 우선순위) | 직접 캠페인에 추가 필요 |
| 중복 광고 상품 | 자동 제외 | 수동 관리 |
| 아이템위너 아닌 상품 | 자동 제외 | 수동 관리 |
| 계정당 개수 | **1개만** | 여러 개 가능 |
| 통제 수준 | 낮음 | 높음 |

**운영 원리**: 머신러닝이 매일 "지금 많은 고객이 찾는 상품"을 발견 → 그 상품에 광고비 집중 → 성과 낮은 상품은 광고비 유지/미지출. 목록은 매일 자동 업데이트.

**20종 탐색 목적 판단**:
- AI스마트: 탐색에 유리, 품절/신규 자동 처리 → 단순하고 편함. 단 계정당 1개 제한
- 매출최적화: SKU별 성과 직접 확인/통제 가능. 품절/신규는 수동 관리

## 현재 상황 (레드콤보 캐리어바퀴)

- 상품: 호환용 캐리어바퀴
- 전체 SKU: 100종 / 현재 입고: 20종 (판매량 상위)
- 고유 ID 체계: gg101, gg102, ... 형식
- 광고 유형: 쿠팡 자동광고
- 월 예산: 100만원

### 논의된 캠페인 구조 옵션

**옵션 A: 캠페인 1개 + 광고그룹으로 나누기**
```
캠페인: 캐리어바퀴_자동_260415
  └─ 광고그룹: 상위 (판매량 상위 10종)
  └─ 광고그룹: 하위 (판매량 하위 10종)
```
- 장점: 데이터 집중, 학습 빠름, 관리 단순
- 단점: SKU별 예산 독립 조절 어려움

**옵션 B: 캠페인 여러 개 (판매량별) + 광고그룹 1개씩**
```
캠페인: 캐리어바퀴_자동_상위_260415 → 상위 1~7위
캠페인: 캐리어바퀴_자동_중위_260415 → 8~14위
캠페인: 캐리어바퀴_자동_하위_260415 → 15~20위
```
- 장점: 캠페인별 예산 독립, 성과 비교 쉬움
- 단점: 캠페인 수 증가, 데이터 분산

**옵션 C: 캠페인 20개 (SKU별)**
```
캠페인: 캐리어바퀴_자동_gg101_260415
캠페인: 캐리어바퀴_자동_gg102_260415
...
```
- 장점: SKU별 예산 완전 독립, 균등 데이터 수집 (탐색 목적에 최적)
- 단점: 관리 복잡, 100종 확장 시 캠페인 100개

### 실제 세팅 현황 (260415 기준)
- 캠페인명: 캐리어바퀴_홍성_260415 (운영 중, 학습 중)
- 광고그룹: 1개 (매출최적화는 캠페인당 광고그룹 1개 고정)
- 등록 상품: 102번 외발휠(운영 중), 101번 더블휠(위너미선정 → 제거 예정)
- 일예산: 100,000원 / 목표 ROAS: 700% (4.16 조정, 쿠팡 제안 280%)
- 주의: 아이템위너 미선정 상품은 광고 노출 안됨 → 등록 제외

### 마진 구조 (캐리어바퀴_홍성)
- 판매가: 11,000원 / 마진: 2,500원 / 마진율: 22.7%
- 손익분기 최소 ROAS: 440% (= 100 ÷ 22.7%)
- 한계 CPC: 147원 (= 마진 2,500원 × 전환율 5.88%)
- ROAS 설정 기준: 440% 이하 → 역마진 / 700% → 현재 운영 / 1000% → 초기 설정

### 네이밍 규칙 (확정)
- 캠페인명: `캐리어바퀴_자동_{구분}_{YYMMDD}`
- 광고그룹명: `캐리어바퀴_{구분}_{YYMMDD}`
- 광고명: `캐리어바퀴_{고유ID}` (예: 캐리어바퀴_gg101)

## 운영 노하우 (출처: 1억맵 강의)

### 광고 1대1의 법칙 (1억맵 직접 명명)

**개념:**
```
광고전환매출순이익금 = 광고전환매출이익금(마진) - 광고비
```
광고비를 썼을 때 내 주머니에 광고비만큼 순이익이 남아야 1:1 달성.

**판단 기준:**
| 상태 | 조건 | 액션 |
|------|------|------|
| 정상 | 순이익금 ≈ 광고비 (1:1) | 유지 |
| 과투자 | 순이익금 < 광고비 | 광고비 절감 |
| 여유 | 순이익금 > 광고비 | 광고비 증액 |

**실제 예시 (3개월):**
- 광고비: 1억 1천만원
- 광고전환매출이익금(마진): 2억 6천만원
- 광고전환매출순이익금: 2억 6천 - 1억 1천 = **1억 5천만원**
- 비율: 1억 5천 / 1억 1천 ≈ **1.32** → 순이익 > 광고비 → 광고비 증액 검토

**핵심 함정: 광고비 올린다고 순이익 비례 증가 안 함**

광고비 절감이 필요한 상황 (순이익 < 광고비)에서:
- ❌ 잘못된 접근: "광고비 2배 올리면 순이익도 2배 나오겠지"
- 이유: 광고비↑ = CPC 단가↑, 클릭률/전환율은 그대로 → 효율 악화

✅ **올바른 접근: 광고비를 ~30% 낮춰 CPC 단가 낮추기**
- 예: 10만원 → 7~8만원으로 낮춤
- CPC 단가 낮아짐 → 같은 클릭률/전환율 기준으로 비율 개선
- 경험상 7~8만원 써서 4.5~5만원 순이익 → 비율이 0.5에서 0.6~0.7로 개선
- 이 방향으로 점진적으로 1:1 목표

**순이익 > 광고비 (여유) 상황에서 증액 방법:**
- 절대 순이익 금액이 늘어나는 방향으로 점진적 증액
- 비율이 낮아지는 건 정상 (두려워하지 말 것)
- 예시:
  ```
  10만원 → 순익 20만원 (2:1)
  15만원 → 순익 22만원 (1.47:1)
  20만원 → 순익 23만원 (1.15:1)
  22만원 → 순익 22만원 (1:1) ← 최적점 도달
  ```

**활용 범위: 광고 건강도 즉시 체크**

쿠팡 정산보고서, 빅셀 등 어떤 툴이든 이 공식 하나로 판단 가능:
```
순이익금 vs 광고비
  → 순이익 < 광고비: 광고비 낮춰라 (과투자)
  → 순이익 > 광고비: 광고비 올려라 (여유)
  → 순이익 ≈ 광고비: 최적점, 유지
```
복잡한 지표 분석 없이 **지금 광고가 돈 먹는지 돈 버는지** 한눈에 판단.

### 광고매출 vs 자연매출 이상적 비율 (1억맵)

**이상적 비율: 5:5 (광고 50 : 자연 50)**
- 광고가 줄어도 자연이 50% 받쳐줌
- 자연이 줄어도 광고가 50% 받쳐줌
- → 어느 쪽 변동에도 매출이 크게 흔들리지 않는 안정적 구조

**8:2여도 문제없음 (광고 많은 경우)**
- 광고비 쓰는데 계속 돈이 남는다면 굳이 줄일 이유 없음
- 매출이 나오고 있는 걸 줄일 필요 없음
- 그 광고매출이 결국 자연매출을 올려주는 기반이 됨

**핵심 원리: 쿠팡은 광고판매/자연판매 구분 없이 랭킹 가점 부여**
- 판매가 발생하면 (광고든 자연이든) 쿠팡 노출순위(랭킹) 판매점수 가점
- 쿠팡은 두 채널을 구분하여 가점을 다르게 배분하지 않음
- 따라서 광고 판매량↑ → 랭킹↑ → 자연매출↑

**결론: 광고비 = 자연매출을 올리기 위한 월세(자릿세)**
- 광고매출로 수익이 안 나는 상황이어도
- 심지어 약간 손해가 나는 상황이어도
- 랭킹을 올리고 지키기 위한 월세 개념으로 광고를 유지해야 함

✅ 광고 끊으면: 매출↓ → 랭킹 점수↓ → 자연매출도↓ → 악순환

**월세 개념 적용 기준: 판매량이 뒷받침될 때만 성립**

| 상황 | 광고비 | 일 판매량 | 월 판매량 | 판단 |
|------|--------|---------|---------|------|
| ❌ 낭비 | 10만원 | 2~3개 | ~90개 | 300만원 내고 90개 → 월세 아님, 전략 재검토 |
| ✅ 발판 | 10만원 | 50개 | ~1500개 | 300만원 내고 1500개 → 랭킹 발판 정당화 |

수익 없어도 판매량이 충분하면 → 한 달 후 랭킹↑ → 그때 수익 기대 가능  
개수 적은데 돈만 쓰는 건 전략이 아님

### 물레방아 비유 (1억맵 핵심 철학)

1억맵은 쿠팡 광고 운영을 **물레방아 돌리기**에 비유함.

```
물레방아 = 쿠팡 광고 시스템
물       = 광고비
```

**초기 (운동에너지 0):**
- 찔끔찔끔 물을 틀면 → 물레방아가 돌지 않음
- 처음엔 **팍! 과감하게** 광고비 투입 필요
- 판매량/랭킹이 0인 상태에서 소액 광고는 효과 없음

**궤도 진입 후 (물레방아가 돌기 시작하면):**
- 운동에너지가 생긴 상태 = 판매량/랭킹이 쌓임
- 이제 수도꼭지(광고비)를 **서서히 줄여서** 1:1 법칙에 맞게 조정
- 물레방아는 자체 운동에너지로 계속 돌아감 (자연매출이 받쳐주기 시작)

**핵심 한 줄: "돌면 잠근다"**
- 초기: CPC 높여서라도, 손해 나도 어거지로 돌린다 (판매량/랭킹 확보)
- 궤도 진입 후: 수도꼭지(광고비) 잠가서 1:1 법칙에 맞게 최적화

```
1단계: 어거지로 돌리기 → 판매량 쌓기, 랭킹 올리기
2단계: 돌기 시작하면 잠그기 → 1:1 법칙 적용, 광고비 최적화
3단계: 자연매출이 받쳐주는 안정 구조
```

**초기 효율 낮음은 당연한 것 → 끄거나 줄이면 안 되는 이유**
- 처음부터 광고 효율이 좋을 수 없음 (물레방아가 막 돌기 시작하는 단계)
- 광고 끄거나 금액 줄이면 물레방아 자체를 돌릴 수 없음
- 버텨야 함

**광고비는 변수가 많음 → 남의 세팅 그대로 복붙 불가**

똑같은 금액으로 돌려도 결과가 다른 이유:
- 상품마다 다름
- 판매가마다 다름
- 심지어 **똑같은 상품을 2개 계정에서 동시에 돌려도 광고비 사용액이 다름**

원인: **광고 돌리는 시점 + 쿠팡 AI 롤링 시스템**
→ 내 상품/계정/시점에 맞게 직접 데이터 보면서 조정하는 수밖에 없음

### 적정 CPC 단가 계산법 (1억맵)

**광고 손해의 근본 원인: 내 상품의 적정 CPC 한계가격을 모르는 것**

**공식:**
```
최대 CPC = 1개 판매 수익 × 전환율
```

**예시:**
- 1개 판매 수익: 5,000원
- 전환율: 10% (클릭 10번 → 1개 판매)
- 최대 CPC = 5,000원 × 10% = **500원**

| CPC | 결과 |
|-----|------|
| 500원 초과 | 광고비 > 수익 → 손해 |
| 500원 | 광고비 = 수익 → 1:1 본전 |
| 500원 미만 | 광고비 < 수익 → 이익 |

**핵심: CPC는 전환율을 보고 한계가격을 파악한다**
- 전환율↑ → 허용 CPC↑ (상품 상세페이지 개선이 중요한 이유)
- 전환율을 알려면 광고를 먼저 돌려봐야 함 → 데이터가 쌓이면 역계산 가능

**CPC 최적화 운영 사이클:**

```
① 광고 돌리기 → 전환율 데이터 확보
② 전환율로 내 CPC 한계가격 역산
   (한계 CPC = 1개 수익 × 전환율)
③ 현재 CPC vs 한계 CPC 비교

  [CPC > 한계가격]          [CPC < 한계가격]
  손해 구간                  수익 구간
  → 광고비 낮춰              → 판매량 확인
    CPC 낮추기                 ↓
                           판매량 양호?
                           + 순익 늘고 있음?
                             → 1:1 법칙 적용
                               광고비 증액 시도
                                 ↓
                           증액 후 순익도 올라가면
                           → 유지 / 추가 증액
                           증액 후 CPC 올라가 문제면
                           → 다시 낮추기
④ 반복
```

**요약:**
- 적정 CPC보다 높다 → 무조건 낮춰라
- 적정 CPC보다 낮다 → 판매량 확인 후 증액 여부 결정

### 실전 사례: 신규 상품 런칭 1달 (1억맵 실제 경험)

**배경:**
- 시장을 부수는 가격으로 진입 (마진 거의 없는 판매가)
- 7.23 런칭, 7.24부터 광고 시작 (초기 540원)
- 전환율 25% (7.29 기준) → 경쟁력 있는 가격이 전환율에 직결된 증거

**물레방아 실전 데이터:**

| 날짜 | 광고비 | 판매수 | CPC |
|------|--------|--------|-----|
| 7.29 | 20만원 | 48개 | ~1,000원 |
| 7.30 | 10만원 | 56개 | ~500원 |
| 8.01 | 7만원 | 63개 | ~300원 |

**해석:**
- 7.24~7.29: 물줄기 팍 열기 → 5일간 광고비 키우며 물레방아 강제 가동
- 7.29: 충분히 돌았다고 판단 → 수도꼭지 잠그기 시작
- 광고비 줄였는데 판매는 오히려 증가 → CPC 낮아지며 효율 개선
- 마진 없는 판매가라 20만원씩 버티는 건 불가능 → 타이밍에 맞게 잠근 것

**이 사례가 증명하는 개념들:**
1. 물레방아 - 돌면 잠근다 ✅
2. 광고비 줄여도 CPC 낮아지면 판매 오히려 증가 ✅
3. 가격 경쟁력 = 전환율 직결 ✅
4. CPC 한계가격 인식 → 버틸 수 없는 타이밍에 잠그기 ✅

**케이스 상세 수치:**
- 시장가: 1만원 초중반
- 진입 판매가: 7,900원 (시장파괴)
- 원가: 3,600원
- 실질 마진: 개당 100~200원 손해 (수수료 포함)
- 초반 전략: 손해 감수하고 전환율 + 판매량 기초 확보
- 3천개 판매량 확보 후 재고 소진 → 추가 발주

**현재 상태 (안정화 후):**
- 광고비: 2~3만원 (보수적)
- 광고 판매: 일 20~30개
- **자연판매: 일 70개 이상**
- 초반 20만원 광고는 시범 케이스라 공격적으로 집행한 것

**핵심 교훈:**
개당 100~200원 손해를 감수하며 판매량/랭킹을 산 결과
→ 지금은 광고 거의 안 써도 자연판매 70개+/일 구조 완성

**최종 현황:**
- 판매가: 12,500원으로 인상 (7,900원 → 12,500원)
- 일 판매량: 광고+자연 합산 100개+
- 전략 완성 아크:
```
7,900원 손해 진입
  → 전환율 25% + 광고 팍 → 물레방아 가동
  → 3천개 판매량 → 랭킹 확보
  → 자연판매 70개+/일 구조 완성
  → 판매가 12,500원 인상
  → 광고+자연 100개+/일, 마진 확보
```
손해를 감수한 초반 투자 → 랭킹/구조 매입 → 가격 올려 수익 실현

**소요 기간: 1달도 안 걸림**

**현재 수익 구조 (12,500원 인상 후):**
- 개당 순이익: 4,500원
- 일 판매량: 100개+
- 일 수익: ~45만원
- 손해 감수 진입 → 한달 이내 안정 수익 구조 완성

**초기 투자 관점:**
- 초기 광고비 100만원 이하는 투자비로 간주하고 써야 함
- 이 투자로 랭킹/구조를 매입하는 것
- 45만원/일 수익 구조가 완성되면 초기 투자비는 며칠 만에 회수

### 쿠팡 AI 롤링 시스템 (1억맵)

**AI 작동 로직:**
```
① 광고비 많이 씀 (물줄기 팍)
   → AI: "투자 의지 있네, 좋은 자리 줘볼게"
         ↓
② 좋은 자리 = 노출↑ → 클릭↑
   + 전환율 25% (시장파괴 가격)
   → AI: "오 잘 팔리네? 회전초밥처럼 돌아가네"
         ↓
③ AI가 좋은 자리 유지 결정
   → 광고비 줄여도 클릭수 유지/증가
         ↓
④ 이 시점에서 수도꼭지 살짝 잠그면
   → AI: "효율 좋은 광고네, 오히려 더 밀어줄게"
   → 좋은 자리에서 안 뺌, 오히려 더 노출
         ↓
⑤ 결과: 돈은 덜 쓰면서 클릭은 더 많이 받는 구조 완성
```

**핵심 순서:**
1. 초반: 돈 + 전환율로 AI를 설득
2. AI가 좋은 자리 배정하면: 수도꼭지 잠그기
3. AI는 효율 좋은 광고를 오히려 더 밀어줌
4. 그 자리가 유지되면서 광고 효율 극대화

**왜 광고비 줄였는데 클릭이 늘었나:**
단순히 CPC가 낮아진 게 아니라, AI가 이미 배정한 좋은 자리가 유지되면서 더 많은 노출을 받게 된 것

### 광고 생성 시 필수 선택 기준 (1억맵 3교시)

**상품 설정: AI 자동 선택 금지 → 수동 설정 필수**

### 전체 전략 큰 틀 (1억맵)

**전환율이 전략의 출발점**
- 전환율 높음 = 시장을 부수는 가격으로 진입한 것
- 전환율 낮음 = 둘 중 하나:
  - 제품 자체에 문제가 있거나
  - 설정 가격이 시장파괴/고전환 가격대가 아닌 것

**전략 순서:**
```
① 시장파괴 가격으로 진입 (전환율 확보)
② 광고비 팍 틀어서 물레방아 가동 (AI 설득)
③ AI가 좋은 자리 배정하면 수도꼭지 잠그기
④ 광고 효율 좋은 상태로 안정화
⑤ 객관적 데이터 분석 (CPC 한계가격, 1:1 법칙 적용)
⑥ 이후 판매가 서서히 올리기
   → 광고 데이터는 변하지만 큰 틀(원칙)은 동일하게 적용
```

**불변의 원칙:** 판매가가 바뀌고 데이터가 바뀌어도
- 전환율로 CPC 한계가격 파악
- 1:1 법칙으로 광고비 조절
- 물레방아 원리로 타이밍 판단

이 세 가지 큰 틀은 변하지 않음

### 의미있는 데이터의 기준 (1억맵)

클릭수가 없으면:
```
전환율 모름 → CPC 한계가격 모름 → 광고비 조절 불가
```

찔끔 쓰면 클릭 2회 → 전환율 50%처럼 튀어도 노이즈, 판단 불가  
팍 쓰면 클릭 200회 → 전환율 22%, 신뢰할 수 있는 데이터

**최소 기준: 100클릭 이상**
- 100클릭 미만 → 전환율 데이터 신뢰 불가, 판단 보류
- 100클릭 이상 → 전환율로 CPC 한계가격 역산 가능

- 클릭 100회 전환율 20% → 20명 구매
- 클릭 1000회 전환율 20% → 200명 구매
- 표본이 클수록 전환율 신뢰도↑ → CPC 한계가격 계산 정확도↑

초반 광고비 팍 투입 = 물레방아 가동 + **데이터 빠르게 사는 것** (두 마리 토끼)

### 최소 ROAS 계산법 (1억맵)

**내 제품의 최소 광고 수익률(ROAS)을 알아야 효율적인 광고셋팅이 가능하다.**

**공식:**
```
최소 ROAS = 판매가 / 개당 판매 이익금 × 110
```
- **왜 110?** 쿠팡 광고비에 부가세 10%가 붙기 때문 (100이 아닌 110)
- 즉, 실제 나가는 광고비 = 설정 광고비 × 1.1

**예시:**
- 판매가 10,000원, 개당 이익금 3,000원
- 최소 ROAS = 10,000 / 3,000 × 110 = **약 366%**
- 의미: ROAS 366% 이하로 내리면 광고비(부가세 포함)가 이익을 초과 → 손해

**주의: 할인쿠폰 적용 시 판매가 기준으로 계산**
- 광고는 판매가 기준으로 돌아감
- 쿠폰 할인을 적용해도 ROAS 계산은 원래 판매가 기준

**초기 셋팅: 최소 ROAS × 1.5배로 시작**
```
이유: 처음엔 광고 효율이 낮으므로 여유 있게 세팅
판매가 20,000원, 이익금 3,000원 예시:
  최소 ROAS = 20,000 / 3,000 × 110 = 약 733%
  초기 셋팅 ROAS = 733% × 1.5배 = 1,100%
```

**정리:**
| 단계 | ROAS 설정 | 목적 |
|------|-----------|------|
| 초기 셋팅 | 최소 ROAS × 1.5배 | 광고비 과다 지출 방지, 여유 있게 시작 |
| 최소 기준 | 판매가 / 이익금 × 110 | 이 이하로 떨어지면 광고할수록 손해 |
| 최적화 후 | 데이터 보며 낮추기 | 1:1 법칙 적용, 전환율 확인하며 조정 |

**⚠️ 쿠팡 실전 적용 핵심 (1억맵 4강 18분):**
쿠팡은 CPC를 직접 입력하는 게 아니라 **목표 광고 수익률(ROAS)을 입력**하는 구조.

**ROAS = 광고 수익률 = 얼마 써서 얼마 팔렸냐**
```
ROAS = 광고전환매출 / 광고비 × 100
예: 광고비 10만원 → 광고전환매출 30만원 → ROAS 300%
```

목표 ROAS 설정값에 따라 쿠팡 AI가 CPC 입찰을 자동으로 맞춰줌:
- ROAS 높게 설정 → AI가 CPC 낮게 입찰 → **물줄기 잠그는 것**
- ROAS 낮게 설정 → AI가 CPC 높게 입찰 → **물줄기 푸는 것**

**결국: 목표 ROAS 숫자 조절 = 수도꼭지 조절**

최소 ROAS를 알아야 물줄기를 마음대로 조절할 수 있는 이유:
- 최소 ROAS 이하로 내리면 → 광고할수록 손해 (한계선)
- 그 선을 알면 → 그 위에서 안전하게 줄였다 풀었다 가능

```
최소 ROAS 계산
    ↓
초기: 최소 ROAS × 1.5배로 셋팅 (보수적 시작)
    ↓
물레방아 가동 후: ROAS 낮추기 (물줄기 풀기)
    ↓
1:1 법칙 적용하며 최적 ROAS 찾기
```

**빈칸 퀴즈로 이해하기 (1억맵 4강 18분)**

> 실제로 내가 **x(1만원, 실제판매가)** 로 팔더라도  
> ROAS 계산엔 **y(2만원, 정상가)** 를 넣어야 한다.  
> **y** 를 넣어줘야 광고가 도는데, **z(733%)** 로 설정하면  
> **a(733%)** 보다 안좋게 나올 확률이 높으니 1.5배 잡고 돌린다.  
> → 1,100% 설정 → 실제 광고 결과로 **b(733%)** 가 나와줘야 손해를 안 본다.

```
x = 1만원 (실제 판매가)
y = 2만원 (정상가 = ROAS 계산 분자)
z = a = b = 733% (최소 ROAS = 본전 기준)
x ≠ y,  z = a = b
```
